### 主要问题

1. **Feature List 只有标题没有描述** — Bot 知道有 WhatsApp 功能，但无法解释具体怎么用
2. **定价表是 2024 年版本** — 可能与最新价格不一致
3. **缺乏结构化 QA 内容** — Bot 需要从长文档中"猜"答案，而非直接匹配
4. **缺少术语定义** — 客户问 "什么是 DID" 时，Bot 找不到简洁定义
5. **缺少行业场景内容** — Discovery Phase 中无法按客户行业推荐方案
6. **缺少话术指导** — Bot 推荐产品时的表达不够贴近销售实际

## 需求资料清单

### 一、话术清单（Sales Scripts）

用途：直接作为高优先级 KB chunk，Bot 回复时可精准引用。

| # | 资料名称 | 内容说明 | 建议格式 |
|---|---------|---------|---------|
| 1.1 | **标准回复话术** | 针对 Top 10 高频问题的官方推荐回答（如"CINNOX 和竞品有什么区别？""你们支持哪些渠道？"） | QA 对格式 |
| 1.2 | **异议处理话术** | 客户说"太贵了""我们已经在用 Zendesk/Freshdesk""我需要考虑一下"等场景的应对方式 | 场景 + 推荐回复 |
| 1.3 | **产品亮点话术（Elevator Pitch）** | 每个套餐（BCC / DC / OCC / CXHub）的 1-2 句核心卖点总结 | 套餐名 + 一句话总结 |
| 1.4 | **追加销售话术** | 从低套餐引导到高套餐的推荐话术（如 DC → OCC 的升级理由） | 场景 + 推荐回复 |

| # | 资料名称 | 内容说明 | 建议格式 |

| 2.4 | **多轮对话场景（2-3 个）** | 完整的真实或模拟客户对话记录 | 对话脚本 |

## 资料格式建议

为了最优化知识库切分效果，建议提供资料时遵循以下原则：

### 优先级排序
```
Excel/CSV（结构化） > Markdown/TXT（纯文本） > PDF > PPT
```

### 具体建议

1. **结构化优先** — 表格类数据用 Excel/CSV，切分精度最高
2. **一个主题一个段落** — 每个功能/问题/话术独立成段，避免长段混合多个主题
3. **QA 对格式最佳** — `Q: xxx` / `A: xxx` 格式的文档，切分后天然就是完整的知识单元
4. **明确标注版本和生效日期** — 便于后续更新替换，也让 Bot 知道信息的时效性
5. **避免纯 PPT** — PPT 提取文本后丢失结构，如有 PPT 内容建议同时提供文字版

### 理想的 QA 对示例

```
Q: CINNOX 支持哪些社交媒体渠道？
A: CINNOX 支持 WhatsApp Business、Facebook Messenger、WeChat Official Account、
   LINE 四大社交媒体渠道，均可在同一个工作台统一管理。所有渠道的对话记录
   会自动归集到客户档案中。

Q: BCC 和 DC 套餐的主要区别是什么？
A: BCC（Basic Contact Center）主要提供语音客服能力，适合只需要虚拟号码和
   基础在线客服的团队。DC（Digital Commerce）在此基础上增加了社交媒体渠道
   整合和专属坐席，更适合电商和在线零售业务。DC 月费 $29/坐席，BCC 免费
   但限 3 个坐席。
```
